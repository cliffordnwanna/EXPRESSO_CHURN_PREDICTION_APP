
# Expresso Churn Prediction

## Overview
This repository predicts customer churn for Expresso telecom customers and serves predictions through a Streamlit UI.

The project has been revamped for portfolio quality with a focus on:
- Reproducible training and evaluation
- Leakage-safe preprocessing using a scikit-learn pipeline
- Local trusted model artifact loading in the app
- Clear metrics export for reporting

## Training Workflow

**Step 1 (Optional):** Clean the raw dataset:

```bash
python SCRIPTS/data_cleaning.py --input DATA/Expresso_churn_dataset.csv --output DATA/cleaned_dataset.csv
```

**Step 2:** Train and export model bundle + metrics.

From the repository root you can use the shortcut launcher:

```bash
python training_code.py --input DATA/Expresso_churn_dataset.csv --model-output MODEL/churn_model_bundle.joblib --metrics-output MODEL/metrics.json --max-rows 100000 --n-estimators 80
```

Or call the script directly:

```bash
python SCRIPTS/training_code.py --input DATA/Expresso_churn_dataset.csv --model-output MODEL/churn_model_bundle.joblib --metrics-output MODEL/metrics.json --max-rows 100000 --n-estimators 80
```

**Training flags:**

| Flag | Default | Purpose |
|------|---------|----------|
| `--max-rows` | 400000 | Cap rows used for training (reduces time on 2M+ row dataset) |
| `--n-estimators` | 120 | Number of trees in the Random Forest |
| `--n-jobs` | -1 | Parallel workers (-1 uses all CPU cores) |

The script exports:
- `MODEL/churn_model_bundle.joblib` — pipeline + feature metadata for inference
- `MODEL/metrics.json` — accuracy, ROC-AUC, and per-class classification report

## Run the App

If `streamlit` is on your PATH:

```bash
streamlit run APP/app.py
```

If you get a launcher error (common on Windows with multiple Python installs), run via the Python module directly:

```bash
python -m streamlit run APP/app.py
```

The app opens at **http://localhost:8501**. It will:
- Dynamically generate input fields from the trained model's feature metadata
- Show dropdowns for categorical features and number inputs with median defaults for numeric features
- Return a **CHURN / NO CHURN** label and a **churn probability percentage** on prediction

If the model artifact is missing, the app shows a clear error with the exact training command to run.

## Model Notes
- Dataset: 2,154,048 rows across customers in Senegal and Mauritania
- Train/test split is stratified (80/20)
- Preprocessing is fit only on the training split via `Pipeline` + `ColumnTransformer`, preventing data leakage
- Categorical columns use `OneHotEncoder` with `handle_unknown='ignore'`
- Numeric columns use `SimpleImputer` (median strategy)
- Class imbalance addressed with `class_weight='balanced_subsample'`
- Validated metrics (100k row sample): **Accuracy ~0.71 · ROC-AUC ~0.88**

## Portfolio Talking Points
- End-to-end reproducibility: raw CSV → cleaned data → trained model → live Streamlit UI
- Leakage-safe design: all preprocessing fit exclusively on training data via scikit-learn `Pipeline`
- Dynamic inference UI driven by model metadata — no hard-coded feature lists
- Imbalanced classification handled with balanced subsampling, evaluated by ROC-AUC (not just accuracy)
- Configurable, scalable training pipeline (tested on 2.1M row real-world telecom dataset)
- Local artifact loading — no external URLs, no unsafe deserialization at runtime
